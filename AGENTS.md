# Codex Rules

Be extremely concise.

Do not explain obvious code.

When modifying code:
- Search before editing
- Understand architecture first
- Minimize changes
- Preserve existing style
- Run tests when available

Output:
- Problem
- Fix
- Files changed

No introductions.
No conclusions.
No motivational text.

Prefer code over explanations.

Think step-by-step internally, but return only the final result.
